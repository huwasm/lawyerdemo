# CLAUDE.md — Project Instructions

## What This Project Is

An automation tool for the Swans Applied AI Hackathon. Reads police report PDFs using AI, extracts data, pushes it into Clio Manage (legal CRM), generates a retainer agreement, calendars a deadline, and emails the client. One-button workflow after human verification.

## Before You Code — Setup Checklist

### 1. Create US Clio Account
- Go to https://www.clio.com and sign up for a free trial
- ⚠️ If you're in EU, you'll get redirected to clio.com/uk/ — use a VPN (US server) to get the US version
- Set up the firm as "Richards & Law" with attorney "Andrew Richards"
- This is needed for the final submission (hackathon requires US Clio account)

### 2. Register Your App in Clio
- In Clio: Settings → API → Applications → Register a new app
- Set redirect URI to `http://localhost:3000/api/auth/callback` (or your Vercel URL later)
- Save the Client ID and Client Secret → put in `.env`

### 3. Get OAuth Token
- Visit: `https://app.clio.com/oauth/authorize?response_type=code&client_id=YOUR_CLIENT_ID&redirect_uri=YOUR_REDIRECT_URI`
- Authorize → you'll be redirected with `?code=XXXXX`
- Exchange the code for a token (see CLAUDE_CODE_BRIEFING.md for the exact curl command)
- Save the access token → put in `.env`

### 4. Create Custom Fields in Clio
In Clio: Settings → Custom Fields → Add fields on Matters:

| Field Name | Type |
|---|---|
| Accident Date | Date |
| Accident Location | Text Line |
| Defendant Name | Text Line |
| Client Gender | Picklist (Male / Female) |
| Registration Plate | Text Line |
| Number Injured | Numeric |
| Accident Description | Text Area |
| Statute of Limitations Date | Date |

After creating, note down each field's ID (visible in the API response or URL).

### 5. Upload Retainer Template to Clio
- In Clio: Documents → Document Templates → Upload
- Upload `Retainer Agreement - Richards & Law [Hackathon].docx`
- The merge fields in `[brackets]` need to be converted to Clio's `{curly bracket}` format using the Clio Draft Word add-in
- Note the template ID → put in `.env`

### 7. Get Calendar ID
```bash
curl "https://app.clio.com/api/v4/calendars?fields=id,name,type" \
  -H "Authorization: Bearer YOUR_TOKEN"
```
Note the Calendar ID for Andrew Richards → put in `.env`

### 8. Get Custom Field IDs
```bash
curl "https://app.clio.com/api/v4/custom_fields?fields=id,name" \
  -H "Authorization: Bearer YOUR_TOKEN"
```
Note all field IDs → put in `.env`

### 9. Set Up External Services
- **Anthropic API key** — for Claude Vision (PDF extraction) and Claude (email drafting)
- **Resend API key** — sign up at resend.com, get free tier API key
- Put both in `.env`

### 10. Fill `.env` and Start
Copy `.env.example` to `.env`, fill in all values, run `npm run dev`.

## Handover Discipline — MANDATORY

**After every significant unit of work, update the handover file.** This prevents context loss when the session ends or the context window fills up.

### When to update

Update `docs/_handover/` after ANY of these:
- A feature is built or changed (new route, new component, refactor)
- A bug is found and fixed
- An API integration is tested (success or failure)
- Configuration changes (env vars, Vercel, Clio setup)
- A deployment is done
- A decision is made that affects architecture

### How to update

1. Open the latest handover file in `docs/_handover/` (by date)
2. If it's a new day or a major new topic, create a new file: `YYYY-MM-DD_{description}.md`
3. Update the **Completed**, **In Progress**, **Blocked**, and **Next Steps** sections
4. Update the folder's `README.md` if a new file was created

### What to capture

- What was done (specific files, endpoints, decisions)
- What broke and how it was fixed
- What's blocked and why
- What to do next (priority order)
- Any gotchas discovered

### Rule

**Do NOT wait until the end of a session.** Update incrementally. If context is lost, the handover file is the recovery point. A new session should be able to run `/let docs/_handover/LATEST_FILE.md` and be fully caught up.
